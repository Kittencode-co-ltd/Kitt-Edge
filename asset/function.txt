========================================================================
เอกสารอธิบายการทำงานและฟังก์ชันหลักของแอปพลิเคชัน Kitt-Edge Mobile
========================================================================

1. ภาพรวมการทำงาน (App Flow)
------------------------------------------------------------------------
แอปพลิเคชันทำงานในรูปแบบ Single Page Application (SPA) ผ่านไฟล์ `index.html` 
โดยมี `mobile-app.js` เป็น Core Logic ที่คอยดึงหน้า (Pages) ต่างๆ เข้ามาแสดงผลตามการนำทาง
- เมื่อเปิดแอป: จะเริ่มจากหน้า Splash Screen -> Auth (ล็อกอิน/สมัครสมาชิก)
- หลังล็อกอิน: เข้าสู่หน้า Dashboard (หน้าหลัก)
- การนำทาง (Navigation): ใช้ Bottom Navigation เพื่อสลับหน้า 5 หน้าหลัก:
  1. Dashboard (หน้าหลัก - สรุปภาพรวมรายวิชา, โจทย์แนะนำ, ประวัติกิจกรรม)
  2. AI Tutor (ผู้ช่วย AI / แชทบอทสอบถามข้อมูล)
  3. Exams (สนามสอบ - รวมข้อสอบชุดต่างๆ และระบบข้อสอบ Adaptive)
  4. Progress (ผลการเรียน - กราฟคะแนน, จุดแข็ง, จุดอ่อน, ระดับความชำนาญ)
  5. Profile (โปรไฟล์ - การจัดการบัญชี, เปลี่ยนรูปโปรไฟล์, จัดการแพ็กเกจ (Subscription))

2. ฟังก์ชันและการทำงานหลัก (Core Functions ใน MobileApp)
------------------------------------------------------------------------
ไฟล์ศูนย์กลางของการทำงานคือ `js/mobile-app.js` ซึ่งมีฟังก์ชันหลักแบ่งเป็นหมวดหมู่ดังนี้:

[การเริ่มต้นและการนำทาง]
- init()
  ฟังก์ชันแรกที่ถูกเรียกเมื่อโหลดหน้าเว็บเสร็จ ทำหน้าที่ผูก Event Listener ของ Navigation 
  และเริ่มกระบวนการเปลี่ยนหน้าไปยัง Splash -> Auth

- setupNavigation()
  ผูก Event Click ให้กับปุ่ม Bottom Navigation ด้านล่าง เพื่อเรียกคำสั่งเปลี่ยนหน้า

- navigate(pageName)
  ดึงไฟล์ HTML ของหน้าที่ต้องการจากโฟลเดอร์ `pages/` (เช่น `pages/dashboard.html`) ผ่าน AJAX (Fetch API) 
  แล้วนำมาระบุ (Inject) ลงใน `<div id="mainContent">` และจัดการแสดง/ซ่อน Header หรือ Bottom Nav ให้เหมาะสม

[การโหลดข้อมูลและการแสดงผลผล (Rendering)]
- loadData()
  ดึงข้อมูลจำลอง (Mock Data) จากระบบ (รวมถึงผู้ใช้, วิชาเรียน, ข้อสอบ, กิจกรรม, และความก้าวหน้า)

- renderDashboard() / renderSubjects() / renderQuestions() / renderActivities()
  จัดทำและแสดงโครงสร้าง HTML (DOM Elements) สำหรับหน้า Dashboard เช่น แถบความคืบหน้าของรายวิชา, การด์ข้อสอบแนะนำ, และสรุปประวัติเข้าเรียน

- renderCamera() / handleChat() / addMessage()
  จัดการระบบ AI Tutor โดยรองรับการส่งข้อความจาก Input และจำลองการพิมพ์ตอบกลับของ AI (Chat System)

- renderExams()
  วาดการ์ดแสดงรายชื่อข้อสอบที่มีในระบบลงในหน้า สนามสอบ พร้อมปุ่ม "เริ่มทำข้อสอบ" และสถานะข้อสอบ

- renderProgress()
  จัดการแสดงผลรายงานผลการเรียนแบบละเอียดยิบในหน้า Progress ได้แก่:
  1. ค่าสถิติต่างๆ (คะแนนเฉลี่ย, เวลาเรียน)
  2. กราฟด้วย Chart.js เส้นคะแนนเฉลี่ยปัจจุบันและทำนายคะแนนอนาคต
  3. ระดับความชำนาญรายวิชา (แถบ ProgressBar)
  4. สร้างรายการจุดแข็ง (Strengths) และจุดอ่อน (Weaknesses) ที่สามารถกดได้เพื่อดูรายละเอียด

[ระบบการดูรายละเอียดและจัดการเหตุการณ์พิเศษ]
- showInsightDetail(item, type) / closeInsightDetail()
  ระบบแสดง Bottom-Sheet Modal (หน้าต่างเด้งจากด้านล่าง) ในหน้า Progress เมื่อกดที่รายวิชาในส่วนของจุดแข็ง/จุดอ่อน
  โดยจะแสดงหัวข้อย่อย (Topics), คะแนนแต่ละหัวข้อ, และปุ่ม "ฝึกทำโจทย์" (เฉพาะจุดอ่อน)

- setupGestures() / handleSwipe()
  รองรับการปัดซ้าย-ขวาบนจอทัชสกรีน เพื่อทำการสลับหน้าซ้ายขวาตามเมนู (เช่น Dashboard <-> AI Tutor <-> Exams)

- setupPullToRefresh() / refreshData()
  รองรับระบบดึงลงเพื่อรีเฟรชข้อมูล (Pull to Refresh) ในกรณีที่อยู่หน้าบนสุดของคอนเทนต์

[ระบบห้องสอบ]
- startExam(examId)
  เปลี่ยนหน้าเข้าสู่ `exam-room` (ห้องสอบเต็มหน้าจอ) พร้อมเตรียมข้อมูลข้อสอบแบบคงที่จาก Mock data

- startAdaptiveExam()
  เปลี่ยนหน้าเข้าสู่ `exam-room` และส่ง Data ชุดข้อสอบพิเศษที่มี Flag `isAdaptive: true` 
  เพื่อให้ตัวระบบห้องสอบ ปรับโหมดเป็นข้อสอบแปรผันความยากตามการตอบแบบเรียลไทม์ (AI Adaptive)

[รูปแบบ Utils (เครื่องมือเสริม)]
- Utils.showLoading() / Utils.hideLoading(): โชว์และประบวงกลมโหลดข้อมูล (Loading Overlay)
- Utils.showToast(message, type): ให้แสดงกล่องข้อความแจ้งเตือนสถานะเล็กๆ รูปแบบ Popup
